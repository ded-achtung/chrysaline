# Chrysaline: Living Information
## Результаты, доказательства, путь к ИИ

> Информация -- не данные. Информация -- живое существо.
> Первый тест: 28 марта 2026. Код движка: ~500 строк Python.
> Ни одного изменения движка между доменами и языками.

---

## 1. Что это

Chrysaline -- система, в которой каждый кусок знания является живым существом.
Существа рождаются, питаются, сливаются, скрещиваются, голодают и умирают.
Из этих шести механизмов вырастает всё: категории, правила, иерархии, ответы на вопросы, генерация текста.

Это не нейросеть. Не LLM. Не граф знаний. Не копия мозга.
Это новая парадигма, рождённая из вопроса: **что если информация сама живая?**

---

## 2. Шесть механизмов

| # | Механизм | Что делает | Аналогия |
|---|----------|-----------|----------|
| 1 | **Рождение** | Новое слово = новое существо (energy=1.0) | Клетка появляется |
| 2 | **Питание** | Повторное слово = существо крепнет (energy += 0.5) | Клетка получает питание |
| 3 | **Слияние** (merge) | "кот" + "ел" + "рыбу" = организм "кот-ел-рыбу" | Многоклеточный организм |
| 4 | **Скрещивание** (crossbreed) | "кот-ел-рыбу" x "кот-ел-мышку" = "кот-ел-$0" {рыбу, мышку} | Вид с вариативностью |
| 5 | **Поглощение** (absorption) | "кот ел сыр" + существует "кот-ел-$0" = $0 += сыр | Новая пища в экосистему |
| 6 | **Смерть** | energy -= 0.005 за тик; energy <= 0 = смерть | Естественный отбор |

Производные режимы (visiting, генерация, ask, explain) работают **через** эти механизмы, не рядом.

```
плюс -> прибавить (мост учителя)
прибавить один -> следующее число (правило)
после пяти -> шесть (факт)
= 5+1 = 6
```

Одна фраза учителя. Цепочка замкнулась.

### 3.5 Отрицание: через учителя

```
Учитель: learn_negation("не")

Результат:
  visit("страус") -> pos: {птица}, neg: {летает}
  ask("кто летает?") -> {воробей, ворона, голубь, орёл}

Страус = птица. Страус НЕ летает. Оба факта сосуществуют. Валентность +1/-1.

## 4. Честные границы

### Что НЕ работает

| Граница | Суть | Почему |
|---------|------|--------|
| **Вычисление** | 4+4=? -> тишина | Слоты независимы. Нет функциональной связи $0+$1=$2 |
| **3+ смысла омонимов** | "коса" -> 6 групп вместо 3 | Общие слова ("из") склеивают контексты |
| **Формальное "почему?"** | ask("почему кит млекопитающее?") -> частично | Нет явного explain(); visiting находит части, не цепочку |
| **Длинные правила (15+ слов)** | Парсер режет на чанки | Нужна агрегация ключевых слов |

### Два класса знаний

```
Класс 1: Декларативные (РАБОТАЕТ)
  Категории, факты, иерархии, перечисления, ассоциации.
  Механизм: рождение -> слияние -> скрещивание -> visiting.

Класс 2: Функциональные (ЧАСТИЧНО)
  Вычисления, преобразования, причинные связи.
  Решается через: учитель + цепочки правил.
  Для полного решения нужны: функциональные связи между слотами.
```

## 5. Путь к ИИ

### 5.1 Что уже есть

Chrysaline -- это **фундамент**, не готовый ИИ. Но фундамент, на котором можно строить.

Что есть сейчас:
- Обучение из сырых данных (tabula rasa)
- Категоризация без кластеризации
- Обобщение без градиентного спуска
- Память без базы данных
- Забывание без garbage collection
- Ответы на вопросы без retrieval
- Генерация без языковой модели
- Перенос между доменами без fine-tuning
- Работа с любым языком (русский, китайский, японский) без знания языка

Всё это из **6 механизмов и ~500 строк кода**.

### 5.2 Что нужно для ИИ

ИИ = система, которая может:
1. Воспринимать новую информацию
2. Организовывать её в знания
3. Отвечать на вопросы
4. Генерировать новое
5. Объяснять свои ответы
6. Учиться от учителя и самостоятельно
7. Масштабироваться

Chrysaline уже делает пункты 1-4 и 7. Пункты 5-6 частично.

### 5.3 Дорожная карта: от прототипа к ИИ

**Этап 1: Укрепление фундамента**
- [ ] Explain() -- явные цепочки объяснений через visiting
- [ ] Авто-синонимы -- "плюс" и "прибавить" в одной позиции -> скрещивание найдёт
- [ ] Чистая омонимия -- min_cluster_size + вес по редкости контекста

**Этап 2: Самостоятельное обучение**
- [ ] Детекция пробелов -- visiting возвращает пусто = "я не знаю"
- [ ] Формулировка вопросов -- система спрашивает учителя: "что такое X?"
- [ ] Активное исследование -- система сама подаёт себе предложения для проверки

**Этап 3: Масштаб**
- [ ] 10000+ предложений без деградации
- [ ] Rust-реализация для скорости (Python -- прототип)
- [ ] Потоковый ввод (реальный текст, не подготовленные предложения)
- [ ] Многоязычный ввод в один мир (русский + английский + китайский)

**Этап 4: Функциональные знания**
- [ ] Связи между слотами: $0 + $1 = $2 (зависимость)
- [ ] Вычисление как цепочка правил (не арифметика, а логический вывод)
- [ ] Причинность: "потому что" как тип связи между организмами

**Этап 5: Интерфейс**
- [ ] Диалог: многоходовый разговор с контекстом
- [ ] Визуализация экосистемы (кто жив, кто сильный, кто умирает)
- [ ] API для интеграции с внешними системами

### 5.4 Ключевая идея: почему это может стать ИИ

Все существующие подходы к ИИ -- это **контейнеры с данными**:

```
Нейросеть:   данные -> матрица весов -> forward pass -> ответ
LLM:         токены -> контекст -> вероятность -> ответ
Граф знаний: факты -> тройки в базе -> запрос -> ответ
```

Во всех случаях есть **мёртвые данные** и **внешний алгоритм**, который их обрабатывает.

Chrysaline переворачивает это:

```
Живая информация: существа САМИ живут, САМИ организуются, САМИ находят ответы.
```

Нет внешнего алгоритма обработки. Есть экосистема.
Знание не хранится -- оно живёт.
Память -- не записи, а сильные существа.
Забывание -- не удаление, а голод.
Понимание -- не вычисление, а родство.

**Это не аналогия. Это архитектура.** И она работает.

### 5.5 Почему 500 строк -- это сила, а не слабость

```
LLM:        175 миллиардов параметров. Не знает "почему".
Chrysaline: 500 строк кода. Знает через семейные связи.

LLM:        переобучение на новых данных = месяцы GPU.
Chrysaline: поглощение нового слова = 1 тик.

LLM:        галлюцинации (уверенно врёт).
Chrysaline: тишина (если не знает -- молчит).

LLM:        чёрный ящик.
Chrysaline: каждый ответ = прослеживаемая цепочка родства.
```

500 строк -- это не "мало кода". Это **правильный фундамент**. ДНК тоже кодируется 4 буквами.

### 5.6 Что нужно прямо сейчас

Три вещи превратят прототип в рабочую систему:

1. **Explain()** -- чтобы система не просто отвечала, а показывала путь.
   Без этого ответы непроверяемы.

2. **Самостоятельные вопросы** -- чтобы система говорила "я не знаю X".
   Без этого она пассивна.

3. **Масштаб 10k+** -- чтобы доказать что абстракции выживают при реальном объёме.
   Без этого всё может быть артефактом малых данных.

Каждая из этих вещей -- маленький шаг. Ни одна не требует нового модуля.
Каждая решается через существа, родство, visiting.

```

Одна структура данных. Шесть механизмов. Вся система.

## 7. Правила проекта

1. **Информация живая.** Не записываем данные -- рождаем существа.
2. **Функции из жизни.** Обучение = рождение + питание. Память = сильные существа. Обобщение = скрещивание. Забывание = голод.
3. **Одна экосистема.** Производные режимы работают через фундамент, не рядом.
4. **Доказывать шагами.** Каждый шаг -- конкретный эксперимент, конкретный тест, конкретные числа.
5. **Не строить новый завод.** Если нужен отдельный модуль -- стоп. Можно ли через существа?
6. **Каждая проблема решалась маленьким шагом.** Категории -- скрещивание. Отрицание -- учитель. Вычисление -- одна фраза. Ни разу не понадобился новый модуль.
