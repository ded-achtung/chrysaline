# Chrysaline: Активное Познание
## Правила, самопроверка, скептицизм, обучение на опыте

> Существо не просто гуляет по родственникам.
> Существо ПОНИМАЕТ, у кого оно в гостях.
> И учится на собственных действиях.

---

## 1. Суть: от пассивного visiting к активному познанию

Chrysaline уже доказала: информация может быть живой. Существа рождаются, питаются, сливаются, скрещиваются и умирают. Из этого вырастают категории, иерархии, ответы на вопросы.

Но всё это -- **пассивное знание**. Система хранит и выдаёт.

Этот документ описывает следующий слой: **активное познание**. Система не просто помнит -- она понимает, применяет, проверяет себя и учится на ошибках.

Четыре грани одного механизма:

```
1. Правила         -- абстракции "что делать", а не только "что бывает"
2. Самопроверка     -- фоновый visiting после каждого действия
3. Скептицизм       -- конкуренция существ, не отдельный фильтр
4. Обучение на опыте -- результаты действий = новые существа
```

Всё через существ. Всё через энергию. Всё через visiting.

**Ключевой принцип:** не новые модули, а новый слой правил жизни существ.

## 2. Доказано экспериментами

Четыре эксперимента, проведённых 30 марта 2026. Ноль изменений в движке между экспериментами.

### 2.1 Эксперимент 1: Рождение правила из наблюдений (7/7)

**Вход:** 30 слов с буквосочетаниями жи, ши, ки, ри, ни. Посимвольно. Ни одного правила на входе.

**Результат:** система сама нашла правило ЖИ-ШИ.

| # | Что нашла | Как |
|---|-----------|-----|
| 1 | ж+и доминирует (67%) | После Ж буква И в 2/3 случаев |
| 2 | ш+и доминирует (67%) | После Ш то же самое |
| 3 | ж·и в 2.5x сильнее ж·а | fed=50 vs fed=20 |
| 4 | ш·и в 3.3x сильнее ш·а | fed=50 vs fed=15 |
| 5 | И -- брат и Ж и Ш | visiting показал связь |
| 6 | Ж и Ш заменяемы | Система нашла их в одном слоте |
| 7 | Абстракция `$0·и` = {ж, к, н, р, ш} | Паттерн найден скрещиванием |

**Вывод:** скрещивание рождает proto-правила из наблюдений. Без нового кода.

### 2.2 Эксперимент 2: Уточнение правила через конкуренцию (6/6)

**Вход:** 70 слов -- те же 30 + 40 слов с "кы", "ры", "ны", "бы", "ды". Ни одного правила на входе.

**Результат:** система САМА разделила слот на три уровня.

```
ЖЕЛЕЗНОЕ ЯДРО (100% верность И):
  ж·и  fed=55  energy=9.9  — НИКОГДА перед Ы
  ш·и  fed=50  energy=9.8  — НИКОГДА перед Ы

ПЕРИФЕРИЯ (бывает и так и так):
  р·и  fed=10  vs  р·ы  fed=35  — верность И: 22%
  в·и  fed=5   vs  в·ы  fed=10  — верность И: 33%
  с·и  fed=5   vs  с·ы  fed=10  — верность И: 33%

АНТИПОД (никогда перед И):
  б·ы, д·ы, м·ы, т·ы — только перед Ы
```

**Ключевые числа:** ядро (ж/ш) в 4.7x сильнее периферии. У ж·ы и ш·ы -- ноль примеров.

**Вывод:** конкуренция энергий сужает правило до истинного ядра. Никакого quality filter не нужно. Система сама отделила закон от тенденции.

### 2.3 Эксперимент 3: Семантическое правило + исключения (7/7)

**Вход:** 10 птиц с описаниями. Кто летает, кто нет. "Страус не летает", "пингвин не летает".

**Результат:** система удерживает правило и исключения одновременно.

```
visit('страус') → pos: {птица, бегает, большой}  neg: {летает}
visit('пингвин') → pos: {птица, плавает, маленький}  neg: {летает}

ask("кто летает?") → 8 летунов, БЕЗ страуса и пингвина
ask("кто птица?")  → ВСЕ 10 птиц, ВКЛЮЧАЯ страуса и пингвина
```

**Вывод:** конкуренция + отрицание (валентность) работают вместе. Система не путает "быть птицей" и "уметь летать". Механизм уточнения правил переносится с орфографии на семантику.

### 2.4 Эксперимент 4: Мост между уровнями (6/6)

**Вход:** посимвольный поток (те же слова) + 5 формулировок правила из книги:
- "жи пишется с буквой и"
- "после ж пишем и"
- "ж и и всегда вместе"
- "в слове жизнь есть жи"
- "жираф пишется через жи"

**Результат:** мост между уровнями возник сам.

```
ДО КНИГИ:
  'ж' не знает книжных слов
  'жизнь' не знает термин 'жи'

ПОСЛЕ 5 ФОРМУЛИРОВОК:
  'ж' братья: [..., вместе, всегда, пишем, после]
  'жизнь' братья: [в, есть, жи, пишется, слове, через]
  'жи' братья: [буквой, ежи, жизнь, жираф, пишется, через]
```

**Ключевые абстракции-мосты (создались САМИ):**

```
$0·пишется·через·жи   $0 = {жизнь, жираф}
$0·пишется·через·ши   $0 = {камыши, машина}
$0·пишется·с·буквой·и  $0 = {жи, ши}
в·слове·$0·есть·жи     $0 = {ежи, жизнь, жираф}
```

**Мост работает в обе стороны:**

```
ПРЯМАЯ:   жизнь → жи → пишется с буквой и  (слово → термин → правило)
ОБРАТНАЯ: жи → жизнь, жираф, ежи            (правило → примеры)
```

**Вывод:** мост между уровнями не нужно строить специально. Он возникает когда данные содержат элементы обоих уровней. Формулировка "жираф пишется через жи" содержит и конкретное слово из наблюдений ("жираф"), и книжный термин ("жи") -- скрещивание создаёт мост автоматически.

**Принцип:** разнообразие формулировок важнее их количества. Одна формулировка живёт изолированно. Пять разных -- находят общих родственников с наблюдениями.

### 2.5 Что доказано четырьмя экспериментами вместе

```
1. Скрещивание рождает proto-правила из наблюдений         (эксп. 1)
2. Конкуренция энергий сужает правило до истинного ядра     (эксп. 2)
3. Отрицание + конкуренция = правило с исключениями         (эксп. 3)
4. Мосты между уровнями возникают через общих родственников (эксп. 4)

Всё на одном движке. Ноль изменений кода. 
Ноль новых модулей. Существующие механизмы.
```

## 3. Скептицизм: конкуренция, а не фильтр

### 3.1 Почему не нужен отдельный модуль оценки

Как система отличает железное правило от шумного паттерна?

Не через "quality filtering", "профиль подтверждений" или "состояния правила". А через **конкуренцию существ**:

```
ж·и  fed=55, конкурент ж·ы = МЁРТВ     → железное правило
к·и  fed=10, конкурент к·ы  fed=35     → слабая тенденция
б·и  = МЁРТВ, конкурент б·ы  fed=20   → анти-правило
```

Одна метрика: соотношение энергии существа и его конкурентов. Ноль новых состояний.

**Доказано экспериментом 2:** 70 слов, система сама разделила ядро/периферию/антипод.

### 3.2 Работает для любого домена

**Русский язык:**
```
ж·и (сильное) vs ж·ы (мёртвое) → "жи с и" железное
к·и (слабое)  vs к·ы (сильное) → "ки" не правило
```

**Природоведение:**
```
"птицы·летают" (сильное) + исключения с валентностью -1 → правило с границами
кит·млекопитающее (сильное) vs кит·рыба (слабое) → кит = млекопитающее
```

**Математика:**
```
2+3=5 (сильное) vs 2+3=6 (мёртвое) → факт железный
"слева·направо" (ослабевшее) vs "сначала·умножение" (окрепшее) → правило сменилось
```

### 3.3 Питание от подтверждения, не от повторения

```
Существо подтверждается сильными → полное питание (+0.5)
Существо встретилось, нейтрально  → слабое питание (+0.1)
Существо противоречит сильным     → не кормится (+0.0)
```

Ложь, повторённая 10 раз, не крепнет -- потому что сильные конкуренты не дают питания.

### 3.4 Доверие = энергия в контексте конкурентов

Не нужно отдельное поле `confidence`. Уверенность -- это соотношение:

```
Существо сильное, конкуренты мертвы      → "уверена"
Существо сильное, конкуренты тоже сильны  → "неоднозначно"
Существо слабое, конкуренты сильнее       → "скорее нет"
Существо слабое, конкурентов нет          → "слышала, не уверена"
```

Ноль новых полей. Энергия + конкуренция = степень уверенности.

## 4. Правила как живые существа

### 4.1 Факт vs правило: не разные типы, а разные режимы

"кот·ел·$0" отвечает на "что бывает?". "жи·пиши·с·$0" отвечает на "что делать?".

Структурно -- одно и то же существо. Разница в контексте визита:
- пришёл с вопросом → читает факт
- пришёл с задачей → применяет правило

Контекст визита определяет режим. Не тип существа.

### 4.2 Два источника правил

**Снизу (наблюдения):** доказано экспериментом 1. Скрещивание находит паттерн → конкуренция уточняет границы. Правило рождается само.

**Сверху (чтение / учитель):** система читает правило → создаётся существо с энергией 1.0 → подтверждается примерами → крепнет.

**Двойное питание:** когда оба источника совпадают, правило крепнет вдвойне. Доказано экспериментом 4: мосты между наблюдениями и книгой возникают через общих родственников.

**Принцип мостов:** мост между уровнями не нужно строить специально. Он возникает когда данные содержат элементы обоих уровней. Разнообразие формулировок важнее их количества.

### 4.3 Естественный отбор правил

```
Правило работает   → подтверждение → питание → энергия растёт
Правило не работает → нет подтверждения → голод → энергия падает
Правило ложное     → конкуренты сильнее → ускоренный голод → смерть
Новое правило лучше → конкуренция → старое умирает, новое крепнет
```

Ложные и устаревшие правила умирают сами. Не нужен отдельный механизм "отказа от правил".

### 4.4 Модификация правил через отрицание

Доказано экспериментом 3:

```
"птицы·летают" + "страус·НЕ·летает" → правило с исключениями
Правило не умерло. Оно приобрело валентность -1 для {страус, пингвин}.
ask("кто летает?") → 8 птиц без страуса и пингвина ✓
ask("кто птица?") → все 10 включая страуса и пингвина ✓
```

### 4.5 Пример: порядок действий в математике

```
Задача: 3 + 2 × (10 - 5) = ?

Правило-существо: "порядок·действий" → {скобки → умножение → сложение}

  Шаг 1: → "сначала скобки" → (10 - 5) → 5
  Шаг 2: → "потом умножение" → 2 × 5 → 10
  Шаг 3: → "потом сложение" → 3 + 10 → 13
```

**Честная граница:** visiting с действием для многошаговых цепочек -- наименее проработанная часть. Для простых правил (жи-ши, подстановка) уже работает. Для сложных (порядок действий) нужны эксперименты. Главное -- не строить скрытый интерпретатор.

## 5. Самопроверка: фоновый процесс

### 5.1 Механизм

Тот же visiting, но запускается автоматически после каждого действия:

```
Система сделала вывод
    ↓
Visiting: результат → сильные конкуренты → правила
    ↓
Есть ли живой конкурент с большей энергией?
    ├── НЕТ → результат кормится
    ├── ДА, слабый → неуверенный результат
    └── ДА, сильный → результат не кормится, сигнал "не уверена"
```

### 5.2 Примеры

**Математика:**
```
Система решила: 3 + 2 × 5 = 25 (слева направо)
Самопроверка → "порядок·действий" (energy 6.0) → "сначала умножение" → 13
"25" vs "13" → конкурент с сильным правилом → "25" голодает, "13" рождается
```

**Русский:**
```
Система написала "жыраф"
Самопроверка → ж·ы не существует, ж·и energy=55 → противоречие
"жыраф" не кормится, "жираф" кормится
```

### 5.3 Не модуль

Самопроверка = visiting + конкуренция. Оба механизма уже существуют. Единственное изменение: visiting запускается ещё и после собственных действий.

## 6. Обучение на собственном опыте

### 6.1 Результат действия = новое существо

```
Система знает: 2+3=5, правило "прибавить = следующее число"
Встречает: 4+4=?

  Применяет правило → 4→5→6→7→8
  Рождается "4+4=8" (energy 1.0)
  Самопроверка → нет конкурентов → слабое питание
  Учитель подтверждает → полное питание → крепнет
```

### 6.2 Обучение на ошибке

```
Система решила: 3 + 2 × 5 = 25
Самопроверка → должно быть 13

  "25" голодает → умирает
  "слева·направо" ослабевает (дало неверный результат)
  "сначала·умножение" крепнет (дало верный)

Следующий раз: 4 + 3 × 2 = ?
  → "сначала·умножение" (сильное) побеждает → 4 + 6 = 10 ✓
```

### 6.3 Замкнутый цикл

```
БЫЛО:   данные → существа → ответы (однонаправленно)
СТАЛО:  данные → существа → действия → результаты → новые существа
         ↑                                                │
         └────────────────────────────────────────────────┘
```

### 6.4 Защита от самообмана

Существо не может кормить само себя:

```
Внешние данные (текст, учитель)  → +0.5
Подтверждение от сильных существ → +0.3
Собственный опыт                 → +0.1
Самоподтверждение                → ЗАПРЕЩЕНО (+0.0)
```

## 7. Единый механизм

Все четыре грани -- один фоновый цикл:

```
        ┌─────────────────────────────────┐
        │     АКТИВНОЕ ПОЗНАНИЕ           │
        │                                 │
        │  1. Данные (текст/книга/опыт)   │
        │           ↓                     │
        │  2. Скептицизм                  │
        │     Конкуренция с сильными      │
        │           ↓                     │
        │  3. Правила                     │
        │     Найти применимую абстракцию │
        │           ↓                     │
        │  4. Действие                    │
        │     Применить через visiting    │
        │           ↓                     │
        │  5. Самопроверка                │
        │     Результат vs конкуренты     │
        │           ↓                     │
        │  6. Обучение на опыте           │
        │     Результат = новое существо  │
        │           │                     │
        │           └──→ (1)              │
        └─────────────────────────────────┘
```

| Грань | Механизм | Статус |
|-------|----------|--------|
| Правила из наблюдений | Скрещивание + слоты | **Доказано** (эксп. 1) |
| Границы правил | Конкуренция энергий | **Доказано** (эксп. 2) |
| Исключения в правилах | Отрицание + конкуренция | **Доказано** (эксп. 3) |
| Мосты между уровнями | Общие родственники | **Доказано** (эксп. 4) |
| Скептицизм | Конкуренция существ | Логически следует, нужен тест |
| Самопроверка | Visiting после действия | Логически следует, нужен тест |
| Обучение на опыте | Рождение из результатов | Логически следует, нужен тест |

## 8. Что НЕ нужно добавлять

### 8.1 Отдельное поле `confidence`

Не нужно. Уверенность = соотношение энергии существа и конкурентов. Уже вычисляется из существующих данных.

### 8.2 Отдельное поле `is_rule`

Не нужно. Контекст визита определяет режим. Та же абстракция может быть фактом или правилом в зависимости от ситуации.

### 8.3 Система "состояний правила"

Не нужна. Эксперимент 2 доказал: энергия + конкуренция даёт ту же информацию.

```
"candidate"          = energy < 2.0, конкурентов мало
"supported rule"     = energy > 5.0, конкуренты мертвы
"weak tendency"      = energy 2-5, конкуренты живы
"exception-bearing"  = валентность -1 для исключений
```

Всё выражается через энергию + конкуренцию + валентность. Уже есть.

### 8.4 Отдельный "quality filter"

Не нужен. Фильтр = конкуренция. Доказано экспериментом 2.

### 8.5 Специальный механизм мостов между уровнями

Не нужен. Мосты возникают сами через общих родственников. Доказано экспериментом 4. Разнообразие формулировок важнее специального кода.

### Шаг 1: Скептицизм (питание от подтверждения)

- [ ] Изменить формулу питания: подтверждение > повторение

## 14. Новые принципы (добавление к правилам проекта)

**Принцип 7: Скептицизм через конкуренцию**

Правило не требует внешней валидации, если его границы выражаются через конкуренцию живых альтернатив. Не фильтр сверху, а отбор снизу.

**Принцип 8: Мосты через общих родственников**

Знания на разных уровнях связываются не через специальный механизм, а через данные, содержащие элементы обоих уровней. Разнообразие формулировок важнее количества.

**Принцип 9: Железность = отсутствие живых конкурентов**

"Железность" правила -- не отдельный тип и не состояние. Это динамическое свойство: существо доминирует / конкурирует / вытесняется. Определяется соотношением энергий, не присвоенным статусом.
